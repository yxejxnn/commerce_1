## CH.2 커머스 과제

---

## Step 1

### 구현 내용
1. Main 및 Product 클래스 생성
2. Product 클래스 상품명, 가격, 설명을 저장할 필드를 작성 
3. Main 클래스에서 Product 객체 생성 
4. List 선언하고 생성한 Product 객체를 추가 
5. 반복문을 사용해 상품 목록을 출력 
6. 0을 입력하면 프로그램이 종료되도록 구현

### 구현 의도
- 상품 정보를 하나의 객체로 관리하기 위해 Product 클래스 생성
- 여러 상품을 저장하기 위해 List를 사용
- Step 1 요구사항에 맞게 상품 목록 출력과 종료 기능만 구현했습니다.

---

## Step 2

### 구현 내용
1. CommerceSystem 클래스 생성
2. Product 목록을 관리할 List productList 필드 작성
3. 생성자를 통해서 상품 목록을 전달받도록 작성
4. Main 클래스에서 CommerceSystem 객체 생성
5. Main 클래스에서 start() 메서드를 호출하도록 작성

### 구현 의도
- step 1에서 Main이 담당하던 실행 흐름을 CommerceSystem 클래스로 분리
- 상품 목록을 Main에서 생성한 뒤, 생성자를 통해 CommerceSystem에 전달하도록 작성
- Step 2에서는 역할을 나누는 구조에 집중했습니다.

---

## Step 3

### 구현 내용
1. Category 클래스와 Customer 클래스 생성
2. Category 클래스에서는 카테고리 이름과 Product 리스트를 관리하도록 필드 생성
3. 카테고리 이름과 Product 리스트를 반환하도록 작성
4. Main에서 카테고리 생성 후 각 카테고리에 맞는 상품 생성 
5. CommerceSystem에서 카테고리 선택, 상품 목록 출력, 상품 상세 보기 기능 구현

### 구현 의도
- Step 3에서는 카테고리별로 상품을 관리하고 조회할 수 있도록 구조를 확장했습니다.

---

## Step 4

### 구현 내용
- 각 클래스 필드를 private으로 변경하고 캡슐화를 적용하여 직접 접근하지 못하도록 설정

### 구현 의도
- getter를 사용해 데이터를 더 안전하게 관리하도록 수정했습니다.
